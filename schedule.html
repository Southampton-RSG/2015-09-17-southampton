Schedule

The following is an outline of what will be taught in the course. Additional notes on the teaching materials will be available shortly before the course and during the course.

Session 1: Building programs with Python
Sunday 21 June, 10:30 AM - 12:00 Noon
In this session, you'll learn good programming practices with an introduction to fundamentals of Python, how to design and build programs more efficiently and the tools and techniques that you can adopt to produce better code. Following topics will be covered:

  • Using libraries
  • Working with arrays
  • Reading and plotting data
  • Creating and using functions
  • Loops and conditionals: for, if, else, ...
  • Defensive programming
  • Using Python from the command line
  • Reference...



Session 2: Version control with Git
Sunday 21 June, 1:00 PM - 2:30 PM
This session will teach you how to solve problems when you are a part of research team sharing code, methods to keep track of multiple versions of the same code or retrieve modifications in source code and an easier way to backup your work via an insight into version control system called Git. Following topics will be covered:

  • Creating a repository
  • Recording changes to files: add, commit, ...
  • Viewing changes: status, diff, ...
  • Ignoring files
  • Working on the web: clone, pull, push, ...
  • Resolving conflicts
  • Open licenses
  • Where to host work, and why
  • Reference...



Session 3: Unit testing
Sunday 21 June, 3:00 PM - 4:30 PM
Does the code developed by you work? Always? Does it do what you think it does? Does it continue to work even after changes are made? Does it continue to work after system configurations or libraries are upgraded? Does it respond properly for full range of input parameters and edge cases? How will it affect your publications?
This session will teach you concepts and techniques of writing good unit test to reduce complexity of your software, improve the predictability of a change, make development faster and produce a clean code that works as expected and produces desired results. You'll learn these skills through a Python unit testing framework called Nose. Following topics will be covered:

  • Introduction to Nose testing framework
  • Writing and executing unit tests
  • Defensive programming
  • Test-Driven development
  • Reference...
